5. Examen Parcial 1: Cálculo de Salario Neto con Retención

Conceptos Cubiertos: Estructuras Condicionales (`Si/Sino`), Cálculo de Horas Extra (1.5x), Retenciones/Impuestos (10%), Variables Flotantes, Pseudocódigo y Diagramas de Flujo.

Descripción: Este examen parcial se enfocó en el desarrollo completo de algoritmos, incluyendo Pseudocódigo y Diagramas de Flujo. El problema central consistía en un caso de estudio para calcular el salario neto de un empleado. El algoritmo debía manejar las horas extra con una tarifa de 1.5 veces el costo normal, y aplicar una retención de impuesto del 10% solo si el salario bruto excedía los B/.750.

Código C Destacado (Cálculo Final):


    // Cálculo final del salario neto en C
    if (salario_bruto_total > 750.0) {
        monto_impuesto = salario_bruto_total * 0.10;
    } else {
        monto_impuesto = 0.0;
    }

    salario_neto_final = salario_bruto_total - monto_impuesto;
    
    // Mostrar resultados detallados
    printf("Salario bruto total: B/.%.2f\n", salario_bruto_total);
    printf("Impuesto aplicado (10%%): B/.%.2f\n", monto_impuesto);
    printf("SALARIO NETO FINAL: B/.%.2f\n", salario_neto_final);

6. Investigación 2: Análisis Comparativo (C vs. Python)

Conceptos Clave: Análisis FODA, Lenguajes de Alto Nivel (Python), Lenguajes de Bajo Nivel (C), Paradigmas de Programación, Velocidad de Ejecución, Consumo de Memoria.

Descripción: Esta asignación consistió en realizar un **Análisis FODA (Fortalezas, Oportunidades, Debilidades, Amenazas)** comparando los lenguajes de programación **C** y **Python**. La investigación concluyó que:

  • **Python** sobresale por su sintaxis simple, su gran cantidad de librerías y su utilidad en áreas como Inteligencia Artificial y Machine Learning. Sin embargo, es más lento y consume más memoria.
  • **C** sobresale por su control directo sobre recursos y hardware, siendo más eficiente en velocidad de ejecución y consumo de CPU, y es la base de muchos otros lenguajes.
